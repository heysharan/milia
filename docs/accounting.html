Odoo Accounting

User Groups

Accounting team hierarchy from botton to top is as follows:

  • Accounting: Invoicing
  • Accounting: Read-only
  • Accounting: Invoicing & Banks
  • Accounting: Bookkeeper
  • Accounting: Administrator
  • Bank: Validate bank account

Other groups involved:

  • Administration: Settings
  • Documents: User
  • Technical: Stock Accounting Automatic
  • User types: Internal User
  • Expense Groups (hierarchy from bottom to top as follows):
    • Expenses: Team Approver
    • Expenses: All Approver
    • Expenses: Administrator

Invoicing user should be in documents user group to access MSME certificates, and technical group for stock accounting to access advance stock accounting configurations on product category.

Bank Accounts

Accounting Administrator can add new bank account under Accounting >> Configuration >> Banks >> Add a Bank Account. User can select Record transactions manually option if don't want to connect bank with software.

ERP auto-creates one bank journal and one account in CoA.


Invoicing user can create bank account under Accounting page on vendor records. User allowed to approve bank accounts can also enable sending money for that account.

Approval of bank accounts adds additional check for certain payment methods.

Debit Notes

Invoicing user can create debit notes from invoice using Debit Note action.

Payments are accessible under Accounting >> Vendors >> Payments. Bookkeeper can print check using highlighted header button.

Chart of Accounts

Accounts are accessible under Accounting >> Configuration >> Accounting >> Chart of Accounts

Multi Currencies

Multi-currencies in database gets enabled when accounting administrator enables 1 extra currency under Accounting >> Configuration >> Accounting >> Currencies.

Accounting administrator can set currency rates manually, or ERP administrator can enable 3rd party services to update exchange rates at specific intervals.


Exchange difference between invoice validation date and payment date will be posted on configured journal and accounts.

TDS

TDS threshold alerts can be enabled by ERP administrator under Accounting >> Configuration >> Settings.

Accounting administrator can configure TDS section under Accounting >> Configuration >> Accounting >> Chart of Accounts.


Accounting read-only user can set expense account on product under Accounting >> Vendors >> Products.

Invoicing user gets alert on bill.

Invoicing user can manually add TDS using highlighted header button. This will mark bill as partially paid.



Reports

Reports are accessible under Accounting >> Reporting

Expenses

ERP Administrator can configure expense app under Expenses >> Configuration >> Settings. Following configurations can be done:

  • Enabling feature to scan receipts
  • Configuring global outstanding payments account

ERP users can access expense app from main dashboard.

Expenses can be done by employee or directly by company. In case of expense done by employee, a vendor bill will be created.

After registering expense, user has to create report, and submit to manager for further approval.

User being team expense approver can approve expenses for following employees:

  • Employee in which user is configured as an expense manager.
  • Employee to whom user is manager.
  • Employee to whose department user is manager.

Invoicing user in expenses all approver group can enable re-invoicing on expense type product.

Petty Cash

Accounting bookkeeper creates journal entry under Accounting >> Accounting >> Journal Entries.

User registers expense, clicks Create Report, and marks as paid by company.

User goes to report, and clicks Submit to Manager.

Manager sets payment method, and clicks Approve.

Manager clicks Post Journal Entries.

Bookkeeper goes to Accounting >> Accounting >> Journal Items, searches for specific account, and clicks Reconcile.


System assigns partial matching number.

Full reconcile number is assigned when the entries are balanced.
