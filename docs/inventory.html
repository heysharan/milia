Odoo Inventory

User Groups

Inventory user groups hierarchy from bottom to top is as follows:

  • Inventory: User
  • Inventory: Administrator

Other groups involved:

  • Quality: User
  • Administration: Settings

Inventory administrator can configure UoMs.

Inventory administrator setups the warehouse.

Inventory user interacts with product records.

Inventory user interacts with transfer orders.

Inventory adjustment is useful in updating opening stock in new ERP systems.

ERP administrator does following configurations in inventory app.

Applications

Inventory administrator can manage UoMs in UoM categories under Inventory >> Configuration >> Units of Measures >> UoM Categories.

Warehouses

Each configuration does following:

  1. Select address, this comes from Contacts application.
  2. Raw material goes for QC so it is received in 3 steps.
  3. Raw material is picked from warehouse and sent to workcenter for production to start. Post production there should be also QC. So manufacturing is done in 3 steps.
  4. Products ready for dispatch comes in clean area for vacuum cleaning, etc. and goes for final packaging so delivery is done in 3 steps.

Locations

Locations are majorly of two types in Odoo:

  1. Physical
  2. Virtual

Warehouse has one main location, keeping it as a parent location other sublocations can be configured. Those can be configured under Inventory >> Configuration >> Warehouse Management >> Locations.

  • MIL/Stock (main location)
  • Rack 1 (child location)
  • Shelf 1 (child location)


Routes, Rules, Operation Types

Rules are of two types:

  • Pull - generally stock requirement in production workcenter, or at customer location
  • Push - generally from/to QC, from packing to delivery, or finished product from workcenter to inventory

Routes is the collection of rules.

Operation types are selectable on rules and following can be configured on them:

  • Type of operation
    • Receipt
    • Delivery
    • Internal Transfer
    • Manufacturing
  • Source Location
  • Destination Location

3-Steps Receipt

Rule Type Operation Type Source Location Destination Location
Pull Receipt Partners/Vendors MIL/Input
Push Internal Transfer MIL/Input MIL/Quality Control
Push Internal Transfer MIL/Quality Control MIL/Stock

3-Steps Manufacturing

Rule Type Operation Type Source Location Destination Location
Pull Internal Transfer MIL/Stock MIL/Pre-Production
Pull Manufacturing MIL/Pre-Production MIL/Post-Production
Virtual Locations/Production is when components in process
Push Internal Transfer MIL/Post-Production MIL/Stock

3-Steps Delivery

Rule Type Operation Type Source Location Destination Location
Pull Internal Transfer MIL/Stock MIL/Packing Zone
Push Internal Transfer MIL/Packing Zone MIL/Output
Push Delivery MIL/Output Partners/Customers

Inventory administrator can enable MTO route under Inventory >> Configuration >> Warehouse Management >> Routes.


Inventory administrator can also unarchive Subcontracting operation type under Inventory >> Configuration >> Warehouse Management >> Operations Types.

Transfer Orders

Inventory user can access three types of transfers (Receipts, Deliveries, Internal) under Inventory >> Operations >> Transfers.

Inventory user can generate lot/serial numbers on receipt order.

Inventory user can partially delivery and create backorder.


Inventory user can do partial/full returns.

Inventory administrator can create e-Waybills from delivery transfer order.

Products

User configures product by setting name, image, various descriptions, product category, track inventory, internal reference, and ensuring correct routes selected.








Inventory user can also set expiration date on products tracked by lots/serials.

Lots/Serials

On receipt order under Inventory >> Operations >> Transfers >> Receipts, inventory user can set/generate serial numbers.




Setting/generating lot numbers comes with additional option for configuring quantity per lot.

Inventory Adjustment

Inventory administrator sets cost on product.

Inventory user can then count inventory and adjust under Inventory >> Operations >> Adjustments >> Physical Inventory.

Reserve/Unreserve

Based on priority, inventory user can do reservation and unreservation of stock.

For instance, below are the two manufacturing orders, MIL/MO/00061 in which components are available and MIL/MO/00062 which has no components. But the latter is the priority.

Inventory user can access the transfer order picking the components from warehouse, and unreserve stock using icon button.



User can come to transfer order picking components for manufacturing order with higher priority, and click button in header Check Availability. This will bring transfer order from Waiting -> Ready stage.

Reordering Rule

Inventory administrator can configure reordering rules on product.


Scheduler executes one time everyday for checking any new procurements to do based on rules configured. Inventory user can execute it manually also under Inventory >> Operations >> Procurement: run scheduler.

Configurations

ERP administrator can do configurations under Inventory >> Configuration >> Settings.
